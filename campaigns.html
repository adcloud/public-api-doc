Campaigns
A campaign is a collection of ad-media and settings that define how a product will be displayed to users on subscribing websites.

A campaign is a collection of ad-media and settings that define how a product will be displayed to users on subscribing websites. In most cases, a campaign will have an expiry date. A campaign is setup to have only one product. However, a product can be related to any number of campaigns.

Params

Param name Description
bidding_enabled
*required

Allow bidding to take place on this Campaign

value: Must be Integer
one of: 0,1
name
*required

The name of the campaign should be unique and self explanatory

value: Must be String
customer_id
*required

The id of the Customer that owns this campaign, important for reports and for identifying a Campaign.

value: Must be Integer
product_id
*required

Relates the Campaign to the specific product (e.g. xyz company car rental), important for reports and for identifying a Campaign

value: Must be Integer
language_id
*required

Determines which Placements the Campaign will run on. If the language is set to German, for instance, it should not run on English-language placements.

value: Must be Integer
start_date
*required

Determine the lifetime of the campaign.

value: Must be Date
format: yyyy-mm-dd
budget_limit
*required

Determines the total budget and therefore the unit limit of the campaign. The number of clicks effected is dependent on the relevant click-price. It indicates how much in total should be expended for the respective campaign

value: Must be Float
unit_price_maximum
*required

The price field is related to the type of billing model and shows the price per unit.

value: Must be Float
type
*required

Determines the billing model of the campaign. [CPC, CPM, CPX, CPX+]

value: Must be Integer
one of: 0,1,2,3,4
delivery_external
*required 		value: Must be Integer
one of: 0,1
delivery_internal
*required 		value: Must be Integer
one of: 0,1
delivery_type
*required

Determines whether the Campaign is supposed to run on its own placement only, the marketplace, or both. The specific delivery destination is determined by the topic selection or by the "My AdPlacements" section.

value: Must be Integer
end_date
optional

The date on which the Campaign should stop running

value: Must be Date
format: yyyy-mm-dd
locations
optional 		value: Must be String
fallback
optional

Should this Campaign make use of Fallback Ads?

value: Must be Integer
keywords
optional

Keywords to associate this Campaign with

value: Must be String
format: keyword,seperated,by,a,comma
frequency_capping
optional

Should this campaign be Frequency Capped?

value: Must be Integer
one of: 0,1
frequency_capping_days
optional

Number of days to apply Frequency Capping for

value: Must be Integer
cookie_lifetime
optional

Number of seconds the Cookie should exist for

value: Must be Integer
cookie_lifetime_view
optional 		value: Must be Integer
one of: 0,1
delivery_boost
optional

Should delivery boost be applied?

value: Must be Integer
one of: 0,1
status
optional

This allows you to monitor the status of your campaign and to modify it where necessary

value: Must be Integer
exclusion_keywords
optional

A set of words that you do not want this Campaign associated with

value: Must be String
format: keyword,seperated,by,a,comma
keyword_lifetime

The number of seconds the keywords should be applied for
