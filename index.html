Campaign Creation Workflow

This paragraph describes the workflow for creating campaigns via the public Adcloud API.

Requirements

  • Valid company/cloud @ adcloud.net
  • OAuth access to the cloud (id and secret)
  • Knowledge or library to retrieve the OAuth access_token via the public API

Note

If you are using ruby, you can use the official adcloud gem. It's a small wrapper for all available features. You just have to set your OAuth id and secret and you are ready to go. Please check https://github.com/adcloud/api-client-ruby

Workflow

1. Create a customer

Within a cloud, every campaign belongs to a unique customer. Each customer has a unique id which needs to be set when submitting the campaign details. You can create a new customer by issuing a POST request with the customers unique name as form parameter to

curl -X POST https://api.adcloud.com/v2/customers?access_token={ACCESS_TOKEN}

Required parameters:

  • customer[name] String Unique customer name

The response should be something similar to:

2. Create the product

In order to create a new product, submit

curl -X POST https://api.adcloud.com/v2/products?access_token={ACCESS_TOKEN}

Required parameters:

  • product[customer_id] Integer customer id
  • product[name] String Unique product name (uniqueness is checked with customer scope)

5. Upload media files for advertisements

The api will auto scale various versions of an uploaded image if it matches one of the default image ratios. Images which do not match any of those ratios and files which cannot be auto-processed will be treated as custom files and are kept as well.

The file upload is actually a file fetch request. The service expects an accessible url to file which should be uploaded. It fetches the file and post-processes it after successful retrieval.

curl -X POST https://api.adcloud.com/v2/media_files?access_token={ACCESS_TOKEN}

Required parameters:

  • mediafile[uploadedfile] String URL from where to fetch the file
  • mediafile[productid] Integer Reference to a product
  • mediafile[adid] Integer Reference to an advertisements

Optional Parameters:

  • media_file[flash] Integer Wether the upload is flash or not
  • media_file[display] Integer Should be upload be treated as display ad

6. Request approval for the campaign

This last step is optional. If you send a request_approval request, the cloud management team will receive an email that a new campaign is waiting for approval and might confirm or reject it. Therefore, only perform this last step if you really want this campaign to go online!

curl -X PUT https://api.adcloud.com/v2/campaigns/{CAMPAIGN_ID}/offer?access_token={ACCESS_TOKEN}

Response:

{
   "_meta":{
       "message":"status updated",
       "status":200,
       "uuid":"93cb6d2fea6538894eac6ffc6e6c85ee"
   }
}

Resources

Advertisements
Ads are the primary representation of what the end-user sees. They are normal Image-Text but can be of other types too

Resource Description
GET https://api.adcloud.com/v2/advertisements List advertisements
GET https://api.adcloud.com/v2/advertisements/:id Show a single advertisement
POST https://api.adcloud.com/v2/advertisements Create a new advertisement
PUT https://api.adcloud.com/v2/advertisements/:id Update an advertisement
DELETE https://api.adcloud.com/v2/advertisements/:id Destroy an advertisement

Campaigns
A campaign is a collection of ad-media and settings that define how a product will be displayed to users on subscribing websites.

Resource Description
GET https://api.adcloud.com/v2/campaigns List campaigns
GET https://api.adcloud.com/v2/campaigns/:id Show a campaign
POST https://api.adcloud.com/v2/campaigns Create a campaign
PUT https://api.adcloud.com/v2/campaigns/:id Update a campaign
PUT https://api.adcloud.com/v2/campaigns/:id/request_approval Set the Campaign Status to Request Approval
PUT https://api.adcloud.com/v2/campaigns/:id/end Set the Campaign Status to Ended
PUT https://api.adcloud.com/v2/campaigns/:id/offer Set the Campaign Status to Offer

Oauth Requests
In order to gain access to the adcloud public api you must have request an access_token via 2 step Oauth.

Resource Description
POST https://api.adcloud.com/v1/oauth/access_token
