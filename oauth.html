Oauth Requests
In order to gain access to the adcloud public api you must have request an access_token via 2 step Oauth.

In order to gain access to the adcloud public api you are requred to provide your registered client_id and client_secret as well as the grant_type (2 step oauth)

Supported Formats

json
>>>

POST https://api.adcloud.com/v1/oauth/access_token

Request an access_token by providing your clientid clientsecret and grant_type

Supported Formats

json

Examples

Request: 
POST https://api.adcloud.com/v1/oauth/access_token?client_id=ev32cc84d7x05189fs27f42d889ac39aff19a2db&client_secret=04dd11832552a5af2e7e46840554292b2a482561&grant_type=none
Response:
{"access_token":"9f7fa33600c9d5704ff64efc13906278f45e836e3c36ee47ee0d840d32118b0b","scope":""}

Params

Param name Description
client_id
*required

The client_id provided by adcloud

value: Must be String
client_secret
*required

Your client_secret provided by adcloud

value: Must be String
grant_type
*required

The grant type, otherwise known as scope

value: Must be String
default: "none"
